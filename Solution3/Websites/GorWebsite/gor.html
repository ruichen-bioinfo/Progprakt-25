Train a GOR Model

Train a GOR model on a protein secondary structure database in seclib format. The system supports GOR I (single-residue statistics), GOR III (pairwise residue information) and GOR IV (pairwise + coupled information). The resulting .mod file is saved server-side and immediately available for prediction and validation.

CB513 dataset: 513 non-redundant protein chains (<25% pairwise identity) derived from PDB via the Hobohm algorithm — the standard benchmark set for secondary structure prediction.

Predict Secondary Structure

Predict the secondary structure of a protein sequence using a trained GOR model. GOR I uses single-residue information theory. GOR III adds pairwise residue interactions within a 17-residue window. GOR IV further incorporates coupled information between residue pairs. GOR V extends GOR IV with evolutionary information from multiple sequence alignments, typically improving Q3 accuracy by 3–5%.

Output:   H = α-helix  |  E = β-sheet  |  C = coil/loop  |  PH/PE/PC = per-residue probabilities (0–9 scale)  |  MT = match to reference + Q3 accuracy

Validate GOR Predictions

Two validation modes:
• CB513 benchmark: run prediction on CB513.fasta sequences (use "Fetch CB513" in the Predict tab), then upload the result here against the built-in CB513 reference. IDs must match exactly (e.g. 154l).
• Custom dataset: upload your own reference file in seclib format (> id / AS seq / SS structure) and select "Upload custom reference" below.
• Single arbitrary sequence: use the Predict tab's "Reference structure" field to get inline Q3 without a seclib file.

Compute Q3 accuracy, SOV score and per-class metrics (QH, QE, QC) by comparing a prediction file against a reference seclib database.
